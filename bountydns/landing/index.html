BountyDNS

An Open Source DNS Server for Bug Bounty Hunters

What is it?

BountyDNS is tool for Bug Bounty Hunters for determining the resolution of DNS payloads. When a DNS domain is injected into a payload, the targeted asset may resolve the DNS name, indicating that the asset may be vulnerable to command injection, Server Side Request Forgery (SSRF), or Local/Remote File Inclusion (LFI/RFI). BountyDNS ships with:

  • A Hackable DNS Server written in python
  • A Web Interface for viewing what remote IP addresses requested DNS records.
  • The ability to run multiple DNS servers with different "Zones".
  • A RESTful API and easy process for generating log term API Tokens.

Technologies Used

BountyDNS leverages the following technologies and stacks:

  • API: FastAPI (Starlette) as an ASGI API
  • API Documentation: Swagger.io & Redoc
  • DNS Server: Python DNS Server written with dnslib
  • Web Interface: Vuejs & TypeScript

Project Status

BountyDNS is currently a Work in Progress (WIP). The core feature of running the DNS server and determining if the target resolved the record are functional. However, the application should not be considered secure enough to deploy publicly.

New Features in the Works

The following features are currently being worked on:

  • Easy deployment via docker and docker-compose
  • Simple provisioning templating, deploying to AWS via Packer, Terraform, and Letsencrypt
  • Instant feedback via WebSockets
  • Option to receive Email Notifications upon resolution
  • An HTTP and HTTPs server to determine the resolution of URLs
  • An SMTP Server to test for SMTP resolution
